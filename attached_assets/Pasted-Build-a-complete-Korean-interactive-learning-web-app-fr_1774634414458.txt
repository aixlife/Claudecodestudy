Build a complete Korean interactive learning web app from scratch.
App name: "Claude Code 마스터 클래스"

==============================================
TECH STACK
==============================================
- React 18 + Vite
- Tailwind CSS
- localStorage (no backend needed)
- Font: Pretendard

--- CHAPTER 1: 입문편 (6 steps) ---

STEP 1-1: 루트 디렉토리에서 시작하기
  Difficulty: 🟢기초 / Time: 약 3분
  Desc: 클로드 코드는 실행 위치가 중요합니다. 루트 디렉토리가 아닌 곳에서 실행하면 프로젝트 구조를 잘못 읽어요.
  Insight: "루트가 아닌 폴더에서 실행 시 — 다른 폴더 접근에 토큰 낭비, 비효율적 작동"
  Think First Q: "클로드를 API 폴더 안에서 실행하면?"
    Options: ["API 관련 파일만 읽을 수 있어요", "더 빠르게 작동해요", "자동으로 루트를 찾아요"]
    Correct: 0
    Explanation: "맞아요. 다른 폴더 접근에 토큰을 추가로 써야 해서 비효율적이에요."
  
  Install guide card first (before prompt):
    Title: "먼저 Claude Code를 설치해요"
    Steps:
      1. claude.ai/download 방문 → [링크 버튼]
      2. Mac / Windows 선택 후 설치
      3. 앱 실행 후 Anthropic 계정 로그인
      4. 터미널을 열고 프로젝트 폴더로 이동
    
  Code block:
    cd ~/내프로젝트폴더
    claude
  
  Tip: "폴더명을 모르겠으면 터미널에서 'ls' 입력 후 폴더 목록을 확인하세요"
  Aha message: "이제 클로드가 프로젝트 전체를 한눈에 볼 수 있어요!"

STEP 1-2: CLAUDE.md 만들기 (/init)
  Difficulty: 🟢기초 / Time: 약 5분
  Desc: /init 명령어 하나로 CLAUDE.md 파일이 생성됩니다. 이 파일은 클로드의 두뇌 — 매 세션마다 자동으로 읽어요.
  Insight: "CLAUDE.md = 프로젝트의 모든 컨텍스트 / 300줄 이하 유지 원칙 (초과 시 토큰 낭비)"
  Think First Q: "CLAUDE.md가 없으면 클로드는 어떻게 될까요?"
    Options: ["매번 처음부터 프로젝트를 파악해야 해요", "더 창의적으로 일해요", "더 빠르게 실행돼요"]
    Correct: 0
    Explanation: "맞아요. CLAUDE.md가 없으면 매 세션마다 모든 파일을 다시 탐색해야 해요."
  
  Code block 1 (label: "1단계 — CLAUDE.md 생성"):
    /init
  
  Code block 2 (label: "2단계 — 내용 추가"):
    방금 만든 CLAUDE.md에 아래 내용을 추가해줘:
    
    # 절대 규칙
    - 프로덕션 DB에 직접 쿼리 금지
    - .env 파일 절대 커밋 금지
    
    # 프로젝트 구조
    (현재 폴더 구조를 트리 형태로 작성)
    
    # 코딩 컨벤션
    - 컴포넌트: PascalCase
    - 유틸리티: camelCase
  
  Aha message: "CLAUDE.md가 있으면 매 세션마다 클로드가 즉시 맥락을 파악하고 시작해요!"

STEP 1-3: Plan Mode 완전 정복
  Difficulty: 🟢기초 / Time: 약 5분
  Desc: Plan Mode는 클로드가 코드를 바로 짜는 대신 계획을 먼저 보여줘요. 검토 후 승인해야 실행됩니다.
  Insight: "플랜 없이 바로 실행 = 엉뚱한 방향으로 코드 대량 생성 위험 / 토큰도 2~3배 낭비"
  Think First Q: "Plan Mode에서 '테스트 작성해줘'를 입력하면?"
    Options: ["바로 테스트 파일을 생성해요", "테스트 계획을 먼저 보여주고 승인을 기다려요", "에러가 나요"]
    Correct: 1
    Explanation: "맞아요. Plan Mode에서는 클로드가 write 권한이 없어서 계획만 세워 보여줘요."
  
  Code block 1 (label: "1단계 — Plan Mode 전환"):
    키보드에서 Shift + Tab 눌러보기
    → 화면 상단이 "Plan Mode"로 바뀌면 성공!
  
  Code block 2 (label: "2단계 — 첫 명령"):
    간단한 할 일 앱을 만들어줘
  
  Code block 3 (label: "3단계 — 플랜 검토 후"):
    이대로 진행해줘
    (또는: 2번 단계를 먼저 하고 나머지를 진행해줘)
  
  Aha message: "플랜을 먼저 보고 승인하는 습관 — 이게 고수와 초보의 차이예요!"

STEP 1-4: Escape & 되감기
  Difficulty: 🟢기초 / Time: 약 3분
  Desc: 클로드가 엉뚱한 방향으로 가고 있다면? Escape로 즉시 멈추세요. 두 번 누르면 이전 프롬프트로 되감기도 돼요.
  Insight: "잘못된 가정 위에 쌓인 코드는 전부 쓸모없음 — 초반에 잡는 게 핵심"
  Think First Q: "클로드의 Thinking Log를 무시하면?"
    Options: ["시간이 절약돼요", "잘못된 가정을 못 잡아서 코드 전체가 틀어질 수 있어요", "아무 문제 없어요"]
    Correct: 1
    Explanation: "맞아요. 클로드가 세운 가정이 틀렸을 때 초반에 잡지 않으면 나중에 전부 다시 해야 해요."
  
  Code block (label: "실습 — 중단 연습"):
    1. 아무 명령이나 입력 (예: 간단한 버튼 만들어줘)
    2. 실행 중간에 Escape 한 번 → 중단 확인
    3. Escape 한 번 더 → 이전 프롬프트로 되감기 확인
  
  Aha message: "Escape는 실수를 0원으로 되돌리는 마법 버튼이에요!"

  Difficulty: 🟢기초 / Time: 약 3분

  Think First Q: "UI 스타일을 전달하는 가장 효율적인 방법은?"

    Correct: 1
    Explanation: "맞아요. 이미지가 가장 정확하고 토큰 효율도 좋아요."

  Aha message: "이미지 드래그앤드롭 — 이걸 아는 사람과 모르는 사람의 결과물이 달라요!"

STEP 1-6: 필수 슬래시 명령어 + 커스텀 명령어

  Difficulty: 🟡실전 / Time: 약 5분
